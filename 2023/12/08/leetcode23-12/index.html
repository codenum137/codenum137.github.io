2008. 出租车的最大盈利

你驾驶出租车行驶在一条有 n 个地点的路上。这 n 个地点从近到远编号为 1 到 n ，你想要从 1 开到 n ，通过接乘客订单盈利。你只能沿着编号递增的方向前进，不能改变方向。

乘客信息用一个下标从 0 开始的二维数组 rides 表示，其中 rides[i] = [starti, endi, tipi] 表示第 i 位乘客需要从地点 starti 前往 endi ，愿意支付 tipi 元的小费。

每一位 你选择接单的乘客 i ，你可以 盈利 endi - starti + tipi 元。你同时 最多 只能接一个订单。

给你 n 和 rides ，请你返回在最优接单方案下，你能盈利 最多 多少元。

**注意：**你可以在一个地点放下一位乘客，并在同一个地点接上另一位乘客。

  • 个人思路

    • dp

      考虑dp[n]表示从地点1到地点n所能获得的最大利益。可以从小到大遍历计算dp[i]，其转移公式为

      dp[i]=max(maxn,dp[startj]+endj−startj+tipj),endj=idp[i]=max(maxn,dp[start_j]+end_j-start_j+tip_j),end_j=i dp[i]=max(maxn,dp[startj​]+endj​−startj​+tipj​),endj​=i

      意为若此时存在以i为目的地的订单，则计算选择该订单的收入和遍历过程中的最大值maxn相比较，保存最大值。

      AC

  • 官解

    • dp+二分

      maxn是没必要的，dp[i-1]一定是遍历到i的最大值，sort.Search就是二分查找。

1631. 最小体力消耗路径

你准备参加一场远足活动。给你一个二维 rows x columns 的地图 heights ，其中 heights[row][col] 表示格子 (row, col) 的高度。一开始你在最左上角的格子 (0, 0) ，且你希望去最右下角的格子 (rows-1, columns-1) （注意下标从 0 开始编号）。你每次可以往 上，下，左，右 四个方向之一移动，你想要找到耗费 体力 最小的一条路径。

一条路径耗费的 体力值 是路径上相邻格子之间 高度差绝对值 的 最大值 决定的。

请你返回从左上角走到右下角的最小 体力消耗值 。

  • 官解

    该题可以抽象为图论模型，每个格子都是一个点，边的权值是高度差。

    • 二分查找

      我们可以将这个问题转化成一个「判定性」问题，即：

2454. 下一个更大元素 IV

给你一个下标从 0 开始的非负整数数组 nums 。对于 nums 中每一个整数，你必须找到对应元素的 第二大 整数。

如果 nums[j] 满足以下条件，那么我们称它为 nums[i] 的 第二大 整数：

  • j > i
  • nums[j] > nums[i]
  • 恰好存在 一个 k 满足 i < k < j 且 nums[k] > nums[i] 。

如果不存在 nums[j] ，那么第二大整数为 -1 。

  • 比方说，数组 [1, 2, 4, 3] 中，1 的第二大整数是 4 ，2 的第二大整数是 3 ，3 和 4 的第二大整数是 -1 。

请你返回一个整数数组 answer ，其中 answer[i]是 nums[i] 的第二大整数。

  • 个人思路

    已知前面数的第二大整数对后面数没有帮助，无论后面数是大于前面数还是小于前面数，除非等于且在前面数和第一大整数之间。

    倒是可以先排序成一个新数组，寻找更大元素，但如何保证是第二大呢。

    看不太懂，但是栈。考虑n个数，我们维护n个长度为3的栈（其实是数组）来记录第一二大的元素。遍历nums，如nums[i]，先将其置入stack[i][0]，然后遍历stack[i]之前的数组，大于stack[i][0]则push。嘶，这好像是暴力枚举，时间复杂度是O(n2)O(n^2)O(n2)。

    Time Limit Exceeded

    • 45/51 cases passed (N/A)

    提示2中提到an ordered data structure，提示1的non-increasing是指数值还是栈的长度

  • 官解

    • 单调栈（non-increasing stack)

      我们可以考虑将原始问题简化为计算数组中每个数字的下一个比它大的数字。为了解决这一问题，我们可以利用「单调栈」算法。该算法的核心思想是利用一个栈来存储数组元素的下标。在遍历数组时，对于每个元素，执行以下操作：如果当前元素大于栈顶元素对应的值，说明找到了栈顶元素的下一个更大的数字，将栈顶元素出栈，并更新结果数组。如果当前元素小于等于栈顶元素对应的值，将当前元素的下标压入栈中，保持栈的单调递减性质。这样，最终结果数组中存储的就是每个数字的下一个比它大的数字（如果存在的话），如果没有更大的数字，则对应位置的值为 -1。

      下面图解是将数值压入栈，而不是下标。

      这样，通过单调栈找到第一大元素。如何寻找第二大元素，类似单调栈，用堆记录从栈中出来的元素，即需要寻找第二大的元素。

    • 双单调栈

      单调栈中的数据就是非递增的，故可以使用第二个单调栈代替最小堆。

    上述题解之所以存的是下标是因为我们需要知道对应位置上的第二大元素是多少res[st2[len(st2) - 1]] = v。

    此处使用数组模拟栈，故将st1的出栈元素按照st1的顺序加入的st2中（这样才是非递增的）较为简单。但若使用其他语言的stack的push和pop操作，可以使用第三个栈来做中转（栈可以调转顺序）。

2697. 字典序最小回文串
